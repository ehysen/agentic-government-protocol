Specification · the heart of AGP

The canonical flow

A government interaction in AGP is one ServiceSession that a constituent agent and a government agent advance together — agent-to-agent — through a status lifecycle. The same five conceptual steps still apply, but each is an operation on the session rather than a separate object. Often the life event itself — a job loss, a new baby, a move — is what starts it.

Step by step#

  1. Discovery#

    read the Catalog

    The constituent agent reads the government agent’s published catalog to learn which services exist, who’s eligible, what’s needed, through which channels, and which capabilities the agency supports. Discovery is read-only and uses no constituent data — it answers “what is available and what would it take?” No session exists yet.

  2. Start & eligibility#

    create → ServiceSession · status: gathering

    The agent creates a ServiceSession for a service and, with the constituent’s consent, assembles the relevant attributes and checks them against the published eligibility rules. The session sits in gathering, and its messages say exactly what is still needed. The estimate is a preview — the determination still belongs to the agency.

  3. Authorization & consent#

    update → ServiceSession · status: ready_to_submit

    The constituent authorizes this agent and consents to specific, scoped data use; the agent attaches a scoped Authorization to the session, moving it to ready_to_submit. This is the named gap. v1 defines where authorization is granted, scoped, verified, and revoked, but defers the cryptographic / identity mechanism to a companion layer.

  4. Submit & delivery#

    submit → status: in_progress → completed

    The agent submits the session to the government agent. The agency of record verifies the authorization, makes the determination, and — instantly where it can — provisions the benefit, driving the session to completed (or requires_intervention for review, or denied). Submit must produce an authoritative Determination. Authority never leaves the agency; the agent’s job ends at a well-formed submit.

  5. Determination, status & recourse#

    read → the session's Determination

    The session now carries the authoritative Determination: the outcome, the award breakdown, an audit trail, the live status, and an explicit correction / appeal path. Because there is one session, the constituent (via their agent or directly) can always re-read it to see, contest, and correct what was done in their name — recourse is part of the protocol.

The control invariant

Across all five steps, one thing never moves: authority. The constituent agent discovers, assembles, asks, and reports. The agency of record decides and provisions. Every AGP message traces back to an authoritative agency — which is what lets a constituent trust the result.
